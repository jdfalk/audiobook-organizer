# iTunes/Apple Devices Sync Investigation Plan

## Problem

iTunes Library.itl files generated by audiobook-organizer:

- ✅ Open cleanly in iTunes 12.13 on Windows (books visible, library not "damaged")
- ❌ Fail in **Apple Devices** (the new Windows app for iPhone sync) at
  **step 3 "Determining tracks to sync"** with
  *"An error has occurred. You will need to restart Apple Devices."*

User has Apple Music subscription, so only **Audiobooks** is checked in the sync
config. They cannot uncheck/re-check audiobooks safely (would wipe books from
phone).

A pre-audiobook-organizer baseline `iTunes Library.itl` exists at
`/Users/jdfalk/Downloads/itunes-investigation/iTunes Library.itl` — this is from
before audiobook-organizer ever touched it.

## Hypothesis Catalog

The 156-byte `mith` track header in `internal/itunes/itl_le_mutate.go`
populates very few fields (TrackID, Size, TotalTime, TrackNumber, Year,
BitRate, SampleRate, DiscNumber, PID). The tracks are tagged with metadata
(name/album/artist/genre/kind/location) via `mhoh` chunks. Likely culprits
Apple Devices cares about that iTunes is more lenient on:

| # | Hypothesis | Test Strategy |
|---|---|---|
| H1 | Composer / Artist field | already eliminated by user |
| H2 | Media Kind / `stik` byte not set in mith → not classified as Audiobook for sync | Test with explicit Media Kind |
| H3 | Bookmarkable flag missing | Test with Bookmarkable set |
| H4 | "Has Video" bit set wrong | Test with explicit zero |
| H5 | Date Added / Date Modified zero | Test with valid Mac-epoch dates |
| H6 | File Folder Count / Library Folder Count zero | Test populated |
| H7 | File Type atom (e.g. `M4B `) missing | Test populated |
| H8 | Album/Artist Persistent IDs in mith reference msdh blocks (album=2, artist=3) that are empty after stripTracks → dangling refs | Test stripped vs. inflated container set |
| H9 | mhoh location URL format (`file://localhost/` vs `file:///`, drive-letter encoding) | Test variants |
| H10 | Sample Rate / Bit Rate zero | Test populated with realistic values |
| H11 | Total Time zero (Apple Devices may divide / use it) | Test populated |
| H12 | Size zero (sync uses to plan transfer) | Test populated |
| H13 | Disabled flag set | Test cleared |
| H14 | Encoding flag in mhoh wrong for ASCII strings (UTF-16 vs UTF-8 vs MacRoman) | Test all three |
| H15 | Persistent ID generated via `crypto/rand` could collide across tracks or with album/artist PIDs | Test with deterministic non-colliding PIDs |
| H16 | Track count in mlth doesn't match actual mith count | Already handled in code; add an explicit test that intentionally mismatches |
| H17 | Tracks present that no playlist (incl. Library master) references | Test with master playlist references |
| H18 | "Audiobooks" smart playlist missing → no items appear in audiobook sync list | Test injecting it |
| H19 | Library Persistent ID changed across writes — Apple Devices may key device sync state on it | Test preserving baseline's Library PID |
| H20 | Application Version / Major+Minor Version mismatch with what Apple Devices expects | Test variants |
| H21 | Music Folder path invalid / Windows-style vs URL format | Test variants |
| H22 | mhoh chunk lengths / padding off-by-one | Already covered by round-trip; verify |
| H23 | Files referenced don't exist on Windows drive at sync time → step 3 stops on first missing | Test with all locations pointing to a single real local file |
| H24 | mith totalLen vs sum of mhoh lengths | Verify in tests |
| H25 | Album/Artist msdh blocks (blockTypes 2/3) emptied by stripTracks lose data referenced by tracks | Test fully-populated container |

## Deliverables

### Phase 1 — ITL Test Generator (priority)

A new Go entrypoint `cmd/itunes-sync-tests/main.go` that produces, under
`/Users/jdfalk/Downloads/itunes-investigation/sync-tests/<NN-name>/`:

- `iTunes Library.itl` — the test variant
- `iTunes Library.xml` — the equivalent XML library (Apple Devices reads this
  on Windows in some configurations)
- `test-info.json` — describes what changed and why (hypothesis ID)
- `result.json` — placeholder (filled in by user/runner)

Each test starts from the user's verified baseline (`iTunes Library.itl`
pre-aborg) and applies one isolated change. We also include "stack" tests
that combine multiple safe changes.

### Phase 2 — Test Index + Runner

- `index.json` listing every test with id, hypothesis, description.
- `bin/sync-tests-windows.ps1` — a PowerShell script the user copies along
  with the generated folder to Windows. For each test it:
  1. Backs up `%USERPROFILE%\Music\iTunes\iTunes Library.itl` to a one-time
     safe location (only on first run).
  2. Copies the test ITL+XML in place.
  3. Launches iTunes via COM (`iTunesLib.iTunesAppClass`), waits for
     `LibraryPlaylist` to populate, queries track count, asserts no error
     dialog — records *"opens-in-iTunes"*.
  4. Closes iTunes, launches **Apple Devices** (via `start
     shell:AppsFolder\AppleInc.AppleDevices_…!App` if package family is
     known, else by Start Menu launch).
  5. Pops a Windows Forms / Out-GridView prompt: **[Worked]
     [Failed-AppleDevices] [Won't-open-iTunes] [Skip]**.
  6. Writes the user's choice plus iTunes COM track count + any error text
     to `result.json` for the test, then advances.
- A summary script that pivots `result.json` files into a CSV pattern
  table (which axis correlates with success/failure).

### Phase 3 — Diagnostic Diff Tool

`cmd/itl-diff` — read two ITLs and print a structural diff (hdfm header,
msdh blocks, track count, first N differing mith fields, mhoh type
inventory). Useful so once the user finds a "works" variant, we can diff it
against a "fails" variant byte-by-field.

## Specific ITL Variants to Generate

Numbered from the pre-aborg **baseline** (test 00). Each subsequent test is a
*single-axis change* unless marked "stacked."

00. `00-baseline-untouched` — exact byte copy of the baseline ITL.
01. `01-roundtrip-only` — decrypt → inflate → deflate → encrypt with no
    payload changes. (Confirms our pipeline is byte-identical-after-roundtrip.)
02. `02-add-1-track-minimal` — add ONE synthetic track using current code path.
03. `03-add-1-track-with-mediakind` — same, but with new Media Kind byte set.
04. `04-add-1-track-bookmarkable` — Bookmarkable flag set.
05. `05-add-1-track-dates-set` — Date Added/Modified populated.
06. `06-add-1-track-filetype` — File Type atom populated.
07. `07-add-1-track-bitrate-samplerate` — non-zero audio params.
08. `08-add-1-track-totaltime-size` — non-zero size + duration.
09. `09-add-1-track-utf8-loc` — Location string in UTF-8 with file:// URL.
10. `10-add-1-track-utf16-loc` — Location string forced UTF-16.
11. `11-add-1-track-windows-path` — bare `W:\path\file.m4b` no URL prefix.
12. `12-add-1-track-fullhouse` — every known mith field + every known mhoh
    type set to plausible values.
13. `13-remove-last-1` — baseline minus 1 (sanity).
14. `14-replace-loc-only` — baseline with one location updated to point at a
    file the user has on Windows; everything else preserved.
15. `15-real-aborg-output` — the production audiobook-organizer-generated
    ITL exactly as it ships today (control: this is the failing case).
16. `16-real-aborg-output-shrunk` — same but truncated to 1 audiobook track.
17. `17-real-aborg-output-with-baseline-libraryid` — production output but
    with Library Persistent ID copied from the baseline.
18. `18-real-aborg-output-no-genre` — strip the Genre mhoh.
19. `19-real-aborg-output-genre-audiobooks` — Genre = "Audiobooks" (plural).
20. `20-real-aborg-output-genre-empty` — Genre = "".
21. `21-real-aborg-output-no-kind` — strip the Kind mhoh.
22. `22-real-aborg-output-kind-podcast` — Kind = "Podcast" (sanity).
23. `23-real-aborg-output-no-location` — strip the Location mhoh.
24. `24-real-aborg-output-loc-no-url-prefix` — Windows path without
    `file://localhost/`.
25. `25-real-aborg-output-windows-backslash-encoded` — backslashes percent-
    encoded vs. literal.
26. `26-real-aborg-output-each-track-has-master-playlist-entry` — ensure
    every track is in the Library/master playlist.
27. `27-real-aborg-output-no-playlists-except-library` — drop Audiobooks
    smart playlist if any.
28. `28-real-aborg-output-with-audiobooks-smart-playlist` — inject one.
29. `29-real-aborg-output-album-artist-msdh-rebuilt` — reconstruct
    blockType=2 (albums) and blockType=3 (artists) so each track's album/
    artist PIDs resolve.
30. `30-real-aborg-output-zeroed-album-artist-pids` — set those PIDs to
    zero in mith (so iTunes auto-generates).
31. `31-real-aborg-output-deterministic-pids` — replace random PIDs with
    deterministic non-colliding ones.
32. `32-real-aborg-output-mediakind-audiobook` — set audiobook media-kind
    flag in mith.
33. `33-real-aborg-output-bookmarkable` — set Bookmarkable.
34. `34-real-aborg-output-with-dates` — populate Date Added.
35. `35-real-aborg-output-with-bitrate-samplerate-totaltime-size` —
    realistic audio metadata.
36. `36-real-aborg-output-fullhouse` — combine 32-35.
37. `37-aborg-1track-pointing-at-real-file` — single track, location = a
    real `.m4b` already on the user's Windows drive (eliminate H23).
38. `38-aborg-1track-from-baseline-album` — clone one of the baseline's
    own working tracks but change the title.
39. `39-aborg-deletes-only-no-adds` — write-back path that only updates
    locations, no inserts. (Tests whether *insertion* is the problem.)
40. `40-aborg-locupdate-only-on-baseline` — apply our location-update
    write-back to the baseline (no inserts at all).

## SQL Tracking

Reflect tests 00-40 into the `todos` table with id `itl-NN-slug`, status
pending → done as ITL is generated, plus a separate `results` table the
PowerShell harness can append to over the wire (or write locally and we
import).

## Out of Scope / Notes

- We won't try to reverse-engineer Apple Devices' internals beyond what
  iTunes COM exposes; the matrix of test variants is the experiment.
- We will NOT write back to the user's real iTunes Library.itl. The
  PowerShell harness only swaps in test ITLs from a separate folder under
  `%USERPROFILE%\sync-tests\` and ALWAYS preserves a one-time backup of
  the user's pre-test library.
- All deliverables go on branch `investigate/itunes-sync-tests` in worktree
  `/Users/jdfalk/.worktrees/itunes-sync-tests`.
