# BOT TASK: FE-10 — Add Frontend Coverage Thresholds to Vitest Config

**TODO ID:** FE-10
**Audience:** burndown bot
**Branch:** `chore/frontend-coverage-thresholds`
**PR title:** `chore(web): add coverage thresholds to Vitest config`

---

## What This Task Does

Adds minimum coverage thresholds (branches: 60%, functions: 60%, lines: 60%,
statements: 60%) to `web/vite.config.ts` (or `vitest.config.ts`) so that the CI
gate fails if coverage drops below these levels.

---

## What NOT to Do

- **Do NOT set** the thresholds higher than the current coverage (check first).
- **Do NOT remove** existing test configuration.
- **Do NOT add** coverage for code that doesn't have tests yet — only configure the
  thresholds.
- **Do NOT use** 100% thresholds — the codebase may not have complete coverage.

---

## Read First

1. Find the Vitest config:

```bash
find /Users/jdfalk/.worktrees/audiobook-eval/web -name 'vite*.config.*' \
  -o -name 'vitest*.config.*' | head -5
```

2. Check if coverage is already configured:

```bash
grep -n 'coverage\|threshold' web/vite.config.ts web/vitest.config.ts 2>/dev/null
```

3. Run coverage once to see current levels:

```bash
cd /Users/jdfalk/.worktrees/audiobook-eval/web
npm test -- --coverage --run 2>&1 | tail -20
```

Note the current lines/functions/branches percentages.

---

## Steps

### Step 1 — Check current coverage levels

Run the coverage command and note the output table. If current coverage is, say:
- lines: 72%
- functions: 65%
- branches: 58%
- statements: 70%

Set thresholds BELOW current levels with 5% margin:
- lines: 65%
- functions: 60%
- branches: 53%
- statements: 65%

Round down to the nearest 5% to give the bot tasks room.

### Step 2 — Update Vitest config

In `web/vite.config.ts` (or `vitest.config.ts`), add the `coverage` block inside
the `test` section:

```ts
test: {
  // ... existing test config
  coverage: {
    provider: 'v8',   // or 'istanbul' — use what's already installed
    reporter: ['text', 'html', 'lcov'],
    thresholds: {
      lines: 65,
      functions: 60,
      branches: 53,
      statements: 65,
    },
  },
},
```

If a `coverage` block already exists, add only the `thresholds` property.

### Step 3 — Install coverage provider if not present

```bash
cd /Users/jdfalk/.worktrees/audiobook-eval/web
# Check if @vitest/coverage-v8 is installed:
grep 'coverage-v8\|coverage-istanbul' package.json
# If not:
npm install --save-dev @vitest/coverage-v8
```

### Step 4 — Verify

```bash
npm test -- --coverage --run 2>&1 | tail -20
```

The test run should pass (current coverage meets the thresholds).

### Step 5 — Commit and open PR

```bash
git checkout -b chore/frontend-coverage-thresholds

git commit -m "chore(web): add coverage thresholds to Vitest config

Adds minimum coverage thresholds (lines/functions/branches/statements)
```
