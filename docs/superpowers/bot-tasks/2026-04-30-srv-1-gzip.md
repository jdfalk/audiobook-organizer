# BOT TASK: SRV-1 — Add Gzip Compression Middleware

**TODO ID:** SRV-1
**Audience:** burndown bot
**Branch:** `feat/server-gzip-compression`
**PR title:** `feat(server): add gzip compression middleware`

---

## What This Task Does

Adds Gin gzip compression middleware (`gin-contrib/gzip`) to the server so that
large JSON API responses and static frontend assets are compressed, reducing
bandwidth and improving load times.

---

## What NOT to Do

- **Do NOT add** gzip to streaming endpoints (SSE) — streaming and gzip are
  incompatible.
- **Do NOT compress** responses smaller than ~1 KB — the gzip overhead exceeds the
  savings.
- **Do NOT add** gzip twice — check if it's already registered.
- **Do NOT change** any business logic.

---

## Read First

1. Check if gzip middleware is already present:

```bash

```

2. Find the router setup in `internal/server/server.go`. Identify where other
   middleware is registered (e.g., `r.Use(cors...)`, `r.Use(logger...)`).
3. Find the SSE endpoint(s) to exclude them:

```bash
grep -n 'SSE\|EventSource\|text/event-stream\|/events\|/stream' \
  internal/server/server.go | head -10
```

## Steps

### Step 1 — Add the dependency

```bash
cd /Users/jdfalk/.worktrees/audiobook-eval
go get github.com/gin-contrib/gzip
```

### Step 2 — Register the middleware

In `internal/server/server.go`, in the router setup block where other middleware is
registered, add:

```go
import "github.com/gin-contrib/gzip"

// In the router setup:
r.Use(gzip.Gzip(gzip.DefaultCompression,
    gzip.WithExcludedPaths([]string{
        "/api/v1/events",   // SSE endpoint — do NOT compress
        "/api/v1/stream",   // adjust to actual SSE path
    }),
))
```

Set `gzip.DefaultCompression` (level 6). Using `gzip.BestCompression` (level 9)
adds CPU cost without meaningful benefit.

### Step 3 — Verify SSE endpoint is excluded

```bash
grep -n '/events\|/stream\|SSE\|event-stream' internal/server/server.go | head -10
```

Make sure all SSE route paths are in the `WithExcludedPaths` list.
