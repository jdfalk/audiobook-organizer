# UOS-04 — Public SDK (`pkg/plugin/sdk`)

**Companion human spec:** §1, §4, §5, §7.

## Branch

```
feat/uos-04-sdk
```

## Goal

Extract the plugin-author-facing types and interfaces into a stable
`pkg/plugin/sdk` import path. Keep all internal-only types in
`internal/operations/registry`. Plugins MUST only import from
`pkg/plugin/sdk` plus the narrowed store interfaces in
`internal/database/iface`.

## Files to add (under `pkg/plugin/sdk/`)

1. `operation.go` — re-exports `OperationDef`, `ResumePolicy`,
   `Priority`, `ActorMode`, `Phase` from registry types as **type
   aliases** (so registry can keep ownership but plugins import a
   stable path). Use `type OperationDef = registry.OperationDef` etc.

2. `reporter.go` — Reporter interface and PhaseReporter interface as
   spec §4. These ARE defined here (the registry implements them);
   the registry imports this package, not vice versa. Move the
   interface definitions out of `internal/operations/registry/reporter.go`
   into here in this PR; registry's `reporter_db.go` becomes the
   private implementation.

3. `capability.go` — `Capability` type and the constants from spec §5.

4. `events.go` — `EventSubscription` struct, `Bus` interface
   (`Publish(ctx, eventName, payload) error`).

5. `plugin.go` — `Plugin` interface, `Migration` struct, `DisableMode`
   enum.

6. `registration.go` — the public `Registry` interface that plugins
   use:
   ```go
   type Registry interface {
       RegisterOp(def OperationDef) error
       EnqueueOp(ctx context.Context, defID string, params any, opts ...EnqueueOption) (string, error)
   }
   ```
   This is a NARROWED view of the internal registry. Plugins cannot
   call `Cancel`, `Shutdown`, etc. — only register and enqueue.

7. `enqueue_options.go` — `EnqueueOption`, `WithParent`,
   `WithActor`, `WithPriority`. Function-option pattern.

8. `errors.go` — sentinel errors plugins are permitted to return:
   `ErrCanceled` (cooperative cancellation), `ErrQuiesced` (clean
   pause for disable), `ErrPluginCapabilityMissing`.

9. `doc.go` — package-level docs explaining the contract for plugin
   authors. Reference spec §1 explicitly.

10. Tests:
    - Compile-only test asserting nothing in `pkg/plugin/sdk/`
      imports `internal/...` (use `go list -deps` in a CI lint).

## Files to edit

1. `internal/operations/registry/types.go` — types defined in registry
   stay; sdk types become aliases pointing to them. Or move to a
   `internal/operations/types/` package with sdk aliasing if cleaner.
   Pick one approach and keep it — do not split arbitrarily.

2. `tools/cmd/oplint/main.go` (NEW) — lint tool that walks any
   directory under `internal/plugins/` and:
   - Verifies the package only imports from `pkg/plugin/sdk`,
     `internal/database/iface`, `context`, stdlib, and `log/slog`.
     Any other internal import is an error.
   - Verifies declared `Capabilities` match observed usage. For v1,
     "observed usage" is approximated by string-matching imports
     (e.g. importing `net/http` without declaring `network.generic`
     is a finding). Real call-graph analysis is vNext.

3. `Makefile` — add `make oplint` target that runs the tool over
   `internal/plugins/...`.

## Hard rules

- The SDK package MUST NOT import any `internal/` package except via
  the type alias indirection — and that indirection itself MUST be
  one-way (registry depends on sdk, not vice versa).
- Plugin authors MUST be able to write a complete plugin with imports
  limited to: `pkg/plugin/sdk`, `internal/database/iface`, stdlib,
  `log/slog`, `golang.org/x/sync/...`. If a plugin needs anything else,
  the SDK is missing a re-export — file a follow-up, do not let
  plugins import internals.
- The `Registry` interface in the SDK is the NARROWED registry; do
  not expose Cancel/Shutdown there.

## Acceptance criteria

- [ ] `go test ./pkg/plugin/sdk/...` passes.
- [ ] `make oplint` passes when run with no plugins (empty input).
- [ ] `make ci` passes.
- [ ] `go list -deps ./pkg/plugin/sdk/...` does not include any
      `internal/` package (use a CI lint to enforce).
