**PR title:** `fix(scanner): replace fmt.Printf with structured logging in tagger`

---

## What This Task Does

Replaces all `fmt.Printf`, `fmt.Println`, `fmt.Fprintf(os.Stderr, ...)`, and bare
`log.Printf` calls in `internal/scanner/tagger.go` (or equivalent tag-writing code)
with the project's structured logger using key-value pairs.

---

## What NOT to Do

- **Do NOT remove** any log lines — only change the function used.
- **Do NOT change** business logic — only the logging calls.
- **Do NOT use** `fmt.Printf` anywhere in the file after this change.
- **Do NOT add** new log lines that don't already exist.

## Steps

### Step 1 — Identify the logger to use

Look at another file in `internal/scanner/` to see the import path and usage:
```bash
head -30 internal/scanner/scanner.go 2>/dev/null || \
  head -30 internal/scanner/*.go 2>/dev/null | head -50
```

If the project uses `log/slog`:
```go
slog.Info("writing tags", "file", filePath, "title", title)
slog.Error("failed to write tag", "file", filePath, "error", err)
```

If the project uses `log.Printf` with a structured format:
```go
log.Printf("tagger: writing tags file=%s title=%s", filePath, title)
```

Match the project's existing style exactly.

### Step 2 — Replace fmt.Printf calls in tagger.go

For each `fmt.Printf(...)` call:
```go
// Before:
fmt.Printf("Writing tags to %s\n", filePath)

// After (slog):
slog.Debug("writing tags", "file", filePath)
```

Map log levels:
- Progress/info messages → `slog.Info` or `log.Printf`
- Errors → `slog.Error` or `log.Printf("ERROR: ...")`
- Debug details → `slog.Debug`

### Step 3 — Remove fmt import if no longer used

After replacing all `fmt.Print*` calls, check if `fmt` is still needed for
non-log purposes (e.g., `fmt.Errorf`, `fmt.Sprintf`). If not, remove the import.
