# BOT TASK: LOG-3 — Replace fmt.Printf with Structured Logging in backup

**TODO ID:** LOG-3
**Audience:** burndown bot
**Branch:** `fix/log-backup-structured`
**PR title:** `fix(server): replace fmt.Printf with structured logging in backup`

---

## What This Task Does

Replaces all unstructured print calls in `internal/server/backup.go` (or the backup
service file) with the project's structured logger.

---

## What NOT to Do

- **Do NOT change** backup logic — only logging calls.
- **Do NOT use** `fmt.Printf` after this change.
- **Do NOT add** new log lines that don't already exist.

---

1. Find the backup file:

```bash
find /Users/jdfalk/.worktrees/audiobook-eval/internal -name 'backup*.go' | head -5
```

2. Check unstructured print calls:

```bash
grep -n 'fmt\.Print\|fmt\.Fprintf.*Stderr' internal/server/backup.go 2>/dev/null
```

---

## Steps

### Step 1 — Find all print calls in backup.go

```bash
grep -n 'fmt\.Print\|log\.Print\b' internal/server/backup.go
```

### Step 2 — Replace with structured logger

Using the same logger style as the rest of `internal/server/`:

```go
// Before:
fmt.Printf("Starting backup to %s\n", destPath)

// After:
slog.Info("starting backup", "dest", destPath)
```

```go
// Before:
fmt.Printf("Backup complete: %d files\n", count)

// After:
slog.Info("backup complete", "files", count)
```

### Step 3 — Clean up imports

Remove `fmt` if no longer needed for non-log purposes.

```bash
cd /Users/jdfalk/.worktrees/audiobook-eval
go build ./...
go vet ./...
```
