# UOS-14 — Delete old `OperationQueue` + endpoints

**Companion human spec:** §11 phase B.

## Branch

```
feat/uos-14-cleanup
```

## Goal

The migration is complete. Delete the v1 system.

## Files to delete

1. `internal/operations/queue.go` (the OLD queue; not the registry).
   Move any helper utilities still in use to
   `internal/operations/registry/` or to a `pkg/operations/util/`
   package.
2. `internal/operations/mocks/mock_queue.go` (regenerate mocks for
   the new `Registry` interface instead).
3. Any `_v1` mirror files left behind by intermediate PRs.
4. `web/src/services/api.ts` — delete `getActiveOperations`,
   `getRecentCompletedOperations`, and the v1-shaped types.
5. v1 routes from `internal/server/server_lifecycle.go`:
   - `/operations/active`
   - `/operations/recent`
   - `/operations` (the old list endpoint)
   - `/operations/:id/status`
   - `/operations/:id/logs` (replaced by `/operations/v2/:id/logs`)
   - `/operations/:id` (DELETE; replaced by `/operations/v2/:id`)
   - `/operations/history`
   Migration: any external caller hitting these gets a 410 Gone with
   a body pointing to the v2 equivalent.

## Files to edit

1. `internal/server/server.go`:
   - Remove `server.queue` field. Replace remaining references with
     `server.opRegistry` (or the SDK Registry interface).
2. `internal/database/iface_ops.go`:
   - Remove obsolete v1-only methods (`GetInterruptedOperations` is
     replaced by registry's `resumeAfterStartup` walking
     `operations_v2`).
3. Tests across `internal/server/...` and `internal/operations/...`:
   - Update or delete tests that exercised the v1 system.
4. `Makefile` — `make oplint` now runs against ALL plugins.

## Hard rules

- This PR may not break any currently-deployed UI surface. Verify by
  running the full Playwright suite against a fresh build.
- Plugin code MUST NOT import any of the deleted packages.
- `make ci` MUST pass.
- Schema migrations are NOT touched. The old `operations` and
  `operation_logs` tables remain but are no longer written to. They
  can be dropped in a follow-up after a verification window.

## Acceptance criteria

- [ ] `go build ./...` succeeds.
- [ ] `make test-all` passes.
- [ ] `make test-e2e` passes (Playwright).
- [ ] Search confirms zero references to `operations.OperationQueue`
      type outside the registry's internal use.

feat(uos): delete v1 OperationQueue + legacy endpoints
